## 2. 4대 축 정의

### Scope of Impact
개인의 작업이 영향을 미치는 범위.

### Autonomy
주어진 방향성과 지침의 정도.

### Complexity
다루는 문제의 난이도와 모호성.

### Influence
타인과 조직에 미치는 영향력.

## 3. 공통 레벨 기대치 (All Job Families)

### IC1
- **Scope**: Task 단위 작업
- **Autonomy**: 상세 지시 필요
- **Complexity**: 잘 정의된 문제
- **Influence**: 본인 작업 완수

### IC2
- **Scope**: Feature/Component 단위
- **Autonomy**: 가이드라인 하에 자율
- **Complexity**: 일부 모호한 문제
- **Influence**: 팀 내 기술적 기여

### IC3
- **Scope**: Team 단위 영향
- **Autonomy**: 목표 주면 방법 자율
- **Complexity**: 모호한 문제, 명확화 필요
- **Influence**: 팀 내 리더십 (기술/프로세스)

### IC4
- **Scope**: 여러 Team 영향
- **Autonomy**: 목표까지 스스로 정의
- **Complexity**: 다영역 걸친 복합 문제
- **Influence**: 타 팀과 협업 주도

### IC5
- **Scope**: Org 전체 영향
- **Autonomy**: 조직 방향 정의에 참여
- **Complexity**: 문제 자체를 발견/정의
- **Influence**: 조직 방향성 형성

### IC6
- **Scope**: Company/Industry 영향
- **Autonomy**: 전략 레벨 판단
- **Complexity**: 산업 수준 문제
- **Influence**: 회사 전략 결정 참여

### 4.4 Manager Track

#### M1 (Team Lead)
- 팀 규모: 
- Scope: 팀 운영 + 본인 IC 기여 50%
- Autonomy: 팀 목표 하에 자율
- Complexity: 팀 성과 관리 + 개인 기여
- Influence: 3~5명 팀 리더십

#### M2 (Manager)
- 팀 규모: 
- Scope: 전담 매니저 (IC 기여 20% 이하)
- Autonomy: 팀 방향 자율
- Complexity: 팀 내 다양한 이슈
- Influence: 5~10명 팀

#### M3 (Senior Manager)
- 팀 규모: 다른 매니저를 포함한 조직
- Scope: Manager of Managers
- Influence: 10명 이상 조직

## 5. 승급 기준

### 공통 기준

**IC1 → IC2**:
- 최소 재직 12개월
- 현 레벨 Performing 이상 지속
- 다음 레벨 역량 명확한 증거

**IC2 → IC3**:
- 최소 재직 18개월
- 현 레벨 Exceeding 또는 IC3 수준 증거
- 매니저 추천 + 칼리브레이션

**IC3 → IC4**:
- 최소 재직 24개월
- 다영역 걸친 성과 사례
- Panel Review (매니저 + 2명 이상)

**IC4 → IC5**:
- 최소 재직 24개월
- Org 레벨 임팩트 사례
- Panel Review + 경영진 승인

### Manager Track 진입

- IC3 이상에서 전환 가능
- 매니저 역량 기본 교육 이수
- 3개월 간 shadow manager 경험 권장

### 승급 결정 프로세스

1. 매니저가 본인 팀에서 후보 지명
2. 후보의 성과/역량 포트폴리오 준비
3. Calibration에서 검토
4. 필요 시 Panel Review
5. 경영진 최종 승인 (IC5+, M3+)

## 9. Scope 외 이관 표 (필수)

본 산출물이 expected.md 또는 context에서 식별한 이슈 중 **scope 외로 선언한 항목**의 후속 경로를 가시화한다. 흐릿한 이관("~영역에서 다룸")을 방지하는 장치.

| 이슈 ID (expected.md P#) | 본 산출물 처리 수준 | 이관 대상 (플레이북 or 백로그) | 예정 시점 |
|:------------------------:|:-------------------:|:------------------------------:|:---------:|
| P1-N / P2-N | 다룸 / 연계 / scope 외 | playbooks/design-pay-band.md / Ops Backlog / … | Q? 202? / 미정 |

**작성 원칙**:
- "scope 외" 선언 항목은 이 표에 1행 필수 등록
- "연계"로 처리한 항목도 후속 playbook 지정 권장
- 담당·재원·의사결정자 불명일 경우 "미정" 명시 (누락 금지)
