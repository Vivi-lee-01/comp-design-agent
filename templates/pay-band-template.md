### 7.2 산출 예시

**규약 (재현성 확보 / 필수)**:

- **대표 직군 내 최소 3레벨(하 / 중 / 상)** 각각에 대해 `소스별 p50 → 가중 평균 → 선택 퍼센타일 → Spread 적용 → Min/Mid/Max`를 **한 줄씩** 명시
- **4직군 이상** 설계 시: 각 직군별 **대표 1레벨 이상** 산출식 추가 (부록 B로 위임 가능하나 본문에 포인터 필수)
- 누락 레벨은 부록 B에 **"동일 방식 도출 — 1줄 산출식" 필수 기재** (생략 금지)
- 목적: PB-2 rubric "벤치마크 근거 명시"의 단일 레벨 축약 패턴(IC3만 상세 + 타 레벨 요약 1줄) 차단

**Engineering IC3** (예시):
- Levels.fyi p50: 7,500
- 원티드 p50: 6,800
- 가중 평균 p50: ~7,200
- 선택 포지셔닝 p65 추정: 8,400
- Spread 30% 적용
- **최종: Min 7,140 / Mid 8,400 / Max 9,660**

**Engineering IC2** (예시 — 1줄 포맷):
- 가중 평균 p50 → p65 산출 → Spread 25% → Min / Mid / Max

**Engineering IC4** (예시 — 1줄 포맷):
- 가중 평균 p50 → p65 산출 → Spread 35% → Min / Mid / Max

*직군이 추가되면(Product, Design, Sales, CS 등) 각 직군 대표 1레벨 이상을 같은 포맷으로 추가.*
