# OKR (Objectives and Key Results)

## 개념

OKR은 **목표(Objective)**와 **핵심결과(Key Results)**의 조합으로 조직/팀/개인의 방향을 정렬하고 성과를 추적하는 프레임워크. 인텔의 Andy Grove가 창안, 존 도어(John Doerr)가 구글에 전파.

## 구조

### Objective (목표)
- **정성적** 서술
- **영감을 주는 (Inspirational)** 내용
- **시간 제한** 있음 (분기/연간)
- 3~5개 이내 권장

**좋은 예**: "모바일 사용자에게 세계 최고의 경험을 제공한다"
**나쁜 예**: "DAU를 20% 올린다" (이건 KR에 가까움)

### Key Results (핵심결과)
- **정량적** 측정 가능
- Objective 달성 여부를 증명
- Objective당 3~5개
- **결과 지표** (활동 아닌)

**좋은 예**: "모바일 앱 평균 별점 4.2 → 4.6"
**나쁜 예**: "앱 리뷰 응답 프로세스 구축" (이건 활동)

## 두 가지 유형

### Aspirational (Stretch) OKR
- **70% 달성이 목표**
- 도전적, 혁신 유도
- 100% 달성하면 "목표가 너무 쉬웠다"는 신호

### Committed OKR
- **100% 달성이 목표**
- 필수 비즈니스 지표
- 미달 시 즉시 action

둘을 섞어 쓰되 **명확히 구분** 필요.

## 도입 시 고려사항

### 장점
- 전사 목표가 팀/개인 목표와 연결됨 (Alignment)
- 분기 주기로 빠른 피드백 루프
- 투명성 (모든 팀의 OKR이 공개되는 경우)
- 집중 (3~5개로 제한)

### 단점/위험
- **평가 연동 시 hack 발생** (쉬운 KR만 세움)
- **활동 중심 KR**이 많아짐 (측정 쉬워서)
- 분기 주기가 너무 짧아 **장기 전략 무시**
- 모두가 OKR 쓰기에 시간을 너무 많이 씀

## 설계 원칙

### 1. OKR은 평가 도구가 아니다
OKR 달성률을 연봉/평가에 직접 연결하지 말 것. 연결하면 **목표 낮춤 유인** 발생.

대안:
- OKR 과정/결과를 평가 참고 자료로만 활용
- 평가는 별도 루브릭(레벨별 역량)으로

### 2. Top-down + Bottom-up 혼합
- 경영진이 회사 OKR 먼저 설정
- 팀은 회사 OKR과 연결된 팀 OKR 자율 설정
- 개인은 선택적 (모든 개인이 OKR 가질 필요 없음)

### 3. 주기
- **분기 OKR**이 표준
- 연간 OKR은 상위, 분기 OKR은 실행
- 월간 체크인, 분기말 리뷰

### 4. KR 품질 기준
좋은 KR 체크리스트:
- [ ] 숫자로 측정 가능한가?
- [ ] 결과 지표인가? (활동/산출물 아님)
- [ ] 달성 여부를 모호함 없이 판단 가능한가?
- [ ] Objective를 진짜로 측정하는가?

### 5. 중단/변경 프로세스
분기 중에도 환경 변화 시 OKR 조정 가능해야 함. **고정된 약속 아님**.

## 흔한 실수

### 1. OKR = 할 일 리스트
KR이 "X 프로젝트 완료", "Y 론칭" 같은 활동 리스트가 됨.
→ 그 활동으로 **어떤 결과**를 얻을지를 KR로 삼아야 함.

### 2. Too many OKRs
팀당 10개 이상 OKR은 집중 불가. **3~5개가 상한**.

### 3. 평가 직결
달성률 = 평가 점수로 쓰면 **목표 낮춤**과 **hack** 발생.

### 4. 분기말에만 관리
분기 시작 때 세우고 분기말에 회고. 중간 3개월은 방치 → 무의미.
→ **월간 체크인** 또는 **격주 리뷰** 필수.

### 5. Alignment 부재
팀 OKR이 회사 OKR과 연결 안됨. "우리 팀 일 열심히 하자" 수준의 OKR.

## 도입 로드맵

### Phase 1 (3개월): Pilot
- 1~2개 팀에서 시범 운영
- 경영진 OKR 먼저 정립
- 주간 체크인 프로세스 설계

### Phase 2 (6~12개월): 확산
- 전 팀 확대
- 도구 도입 (노션, Lattice, Ally 등)
- 코칭/교육 제공

### Phase 3 (1년+): 정착
- OKR 리뷰 문화 정착
- 평가와의 관계 명확화
- 분기 회고를 통한 프로세스 개선

## MBO와의 차이

| 항목 | MBO | OKR |
|------|-----|-----|
| 주기 | 연간 | 분기 (+ 연간) |
| 목표 설정 방식 | Top-down 중심 | Top-down + Bottom-up |
| 공개성 | 주로 상사-부하 간 | 전사 공개 |
| 평가 연동 | 강함 | 약함 (권장) |
| 목표 난이도 | 100% 달성 전제 | 70% 달성(stretch) + 100% 달성(committed) |
