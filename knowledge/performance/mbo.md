# MBO (Management by Objectives)

## 개념

Peter Drucker가 1954년 *The Practice of Management*에서 제시. **목표 달성을 통한 관리** 철학.

## 핵심 원리

1. **상사와 부하가 합의한 목표**를 설정
2. 목표는 **측정 가능**하고 **시간 제한** 있음
3. 평가는 **목표 달성 여부**로 수행
4. 자기 통제와 자기 동기부여 유도

## 구조

### 목표 특성 (SMART)
- **S**pecific (구체적)
- **M**easurable (측정 가능)
- **A**chievable (달성 가능)
- **R**elevant (관련성 있음)
- **T**ime-bound (시간 제한)

### 운영 주기
일반적으로 **연간**. 반기/분기 변형도 있음.

```
연초: 목표 합의
중간: 진행 점검
연말: 달성 평가 → 보상/승급 결정
```

## OKR과의 차이

| 항목 | MBO | OKR |
|------|-----|-----|
| 주기 | 연간 | 분기 |
| 목표 수 | 많을 수 있음 | 3~5개 제한 |
| 설정 방식 | Top-down | Top-down + Bottom-up |
| 공개성 | 상사-부하 간 | 전사 공개 |
| 난이도 | 100% 달성 전제 | 70% stretch + 100% committed 혼합 |
| 평가 연동 | 강함 (직접) | 약함 (분리 권장) |

## 장점

- 오래된 프레임워크로 **안정성** 검증
- 대기업/전통 산업에 **적합**
- 목표 설정이 명확하면 **측정 쉬움**
- 평가 근거가 명확

## 단점

- **Top-down 편향**: 상사가 주도하면 합의가 형식화
- **평가 직결의 부작용**: 달성 쉬운 목표만 세우는 유인
- **유연성 부족**: 연간 주기는 변화 빠른 환경에 부적합
- **활동 중심화**: "X 프로젝트 완료" 같은 활동이 목표가 됨
- **팀 간 연결 약함**: 개인 목표에 치우침

## MBO가 적합한 상황

- **안정적 사업 환경** (연간 계획이 의미 있음)
- **대기업/전통 산업**
- **성과가 명확히 측정 가능한 직무** (영업, 제조 등)
- **경영진이 성과 중심 문화 원함**

## MBO가 부적합한 상황

- 빠르게 변하는 테크/스타트업
- 장기 R&D, 창의 작업
- 팀 협업이 핵심인 작업 (개인 목표로 쪼개기 어려움)

## 개선된 MBO

현대적 버전은 OKR과 유사해짐:
- 분기 사이클
- Stretch goal 허용
- 평가 연동 완화
- 공개성 증가

사실상 **OKR이 MBO의 진화형**이라고 볼 수 있다.
