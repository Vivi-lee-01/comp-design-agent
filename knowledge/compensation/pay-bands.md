# Pay Bands (페이밴드) 설계

## 개념

페이밴드는 **특정 레벨/직무/지역 조합에 대한 급여 범위**를 정의한 것. 같은 밴드에 속한 사람은 유사한 급여 범위 내에서 보상받는다.

## 기본 구조

각 밴드는 **Min / Midpoint / Max** 3개 값으로 정의:

```
Band Spread = (Max - Min) / Midpoint

일반적 범위:
- IC 초급(IC1-IC2): 20~30% spread
- IC 중급(IC3-IC4): 30~40% spread
- IC 시니어(IC5+): 40~50% spread
- Manager/Director: 40~60% spread
```

상위 레벨일수록 spread가 커지는 이유: 역할 내 편차가 크고, 시장 변동성도 크기 때문.

## 설계 단계

### Step 1: Market Anchor 결정

시장에서 어떤 포지션을 잡을지 결정.

| Position | 의미 | 적합한 상황 |
|----------|------|-------------|
| p50 (중위값) | 시장 평균 | 안정적 중견 기업 |
| p60~p65 | 시장 중상위 | 성장 중인 스타트업 |
| p75 | 시장 상위 25% | 인재 경쟁 치열한 영역 |
| p90 | 시장 최상위 | 특정 핵심 포지션 |

**주의**: 전 직군 동일 포지션보다 **직군별 차등**이 현실적. (예: 엔지니어 p75, 오퍼레이션 p50)

### Step 2: Midpoint 산정

각 레벨의 midpoint = 시장 벤치마크 데이터의 선택한 percentile 값.

**데이터 소스**:
- 국내 IT: 원티드 인사이트, 크레딧잡, 잡플래닛
- 글로벌 IT: Levels.fyi, Glassdoor, Radford (유료)
- 전통 산업: 머서(Mercer), 윌리스타워스왓슨 (유료)

**가공 필요**:
- 스톡옵션 포함/제외 분리
- 본인/경력직 구분
- 도시/지역 구분

### Step 3: Spread 결정 및 Min/Max 산출

```
Midpoint이 5,000만원, spread 30%:
Min = 5,000 × (1 - 0.15) = 4,250만원
Max = 5,000 × (1 + 0.15) = 5,750만원
```

일반적으로 mid 기준 ±15~25% 범위로 설정.

### Step 4: Overlap 설계

인접 레벨 간 **밴드 중첩**을 얼마나 둘지 결정.

```
Level 3: 4,250 ~ 5,750 (mid 5,000)
Level 4: 5,100 ~ 6,900 (mid 6,000)
       ↑ overlap: 5,100 ~ 5,750
```

- **Overlap 없음**: 명확하지만 승급 시 급격한 점프 발생
- **Overlap 있음**: 유연하지만 레벨 차이 의미 약화

일반적으로 **20~30% overlap** 권장.

### Step 5: Location Factor 적용 (선택)

지역별 조정 계수 적용.

```
Base (서울 기준): mid 5,000만원
부산 factor 0.9: mid 4,500만원
원격(지방) factor 0.85: mid 4,250만원
```

**주의**: Location factor는 **공정성 논쟁**이 크다. 최근 원격근무 확산으로 "같은 일 = 같은 돈" 원칙으로 location factor 폐지하는 기업도 증가.

### Step 6: In-band Positioning 규칙

밴드 안에서 **어느 지점**에 위치시킬지 결정하는 규칙.

GitLab 방식 (Compa Group):
- **Learning** (Min ~ Mid-5%): 해당 레벨 역량 학습 중
- **Performing** (Mid-5% ~ Mid+5%): 해당 레벨 역량 충족
- **Exceeding** (Mid+5% ~ Max): 해당 레벨 역량 초과

이 방식의 장점: 개인 급여 결정이 **역량 평가**와 연결됨. "왜 A가 B보다 많이 받나"에 답할 수 있음.

## 검증 체크리스트

설계한 밴드를 아래 기준으로 검증:

- [ ] 모든 현직자가 밴드 범위 내에 위치하는가? (벗어나면 조정 필요)
- [ ] 인접 레벨 간 overlap이 설계 의도대로인가?
- [ ] 같은 레벨 내 **남녀 평균 급여 차이**가 유의미한가? (페이 갭 점검)
- [ ] 신입/외부 영입 시 현직자와 **역전 현상**이 없는가?
- [ ] 매니저 M1과 시니어 IC(IC5)의 밴드가 **dual track** 원칙대로인가?

## 흔한 실수

### 1. 밴드를 "현재 급여 분포"로 역산
현 급여를 그대로 받아들여 밴드를 만들면, 기존 불평등이 고착화됨.
**올바른 접근**: 시장 벤치마크 먼저 → 밴드 설계 → 현직자 매핑 → 이탈자 조정 계획.

### 2. Spread를 너무 좁게
Spread가 10~15%면 성과 차등이 불가능. 고성과자가 금방 max에 닿고 동기부여 상실.

### 3. Location Factor 과다 적용
5단계 이상의 location factor는 운영 부담이 크고 공정성 이슈 발생. **2~3단계**가 실용적.

### 4. 밴드 갱신 주기 부재
시장 변동 반영 없이 3~5년 쓰면 전면 재설계가 필요해짐. **연 1회 갱신**이 표준.

### 5. 밴드를 비밀로 운영
밴드 자체를 숨기면 **투명성 원칙 위반**. 최소한 본인이 속한 밴드의 min-max는 공개 권장.
