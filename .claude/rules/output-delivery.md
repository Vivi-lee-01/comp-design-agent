# 산출물 전달 규율 — 후속 포맷 제안 의무

본 문서는 모든 평가보상 설계 플레이북의 **산출물 마감 시점**에 Claude가 지켜야 할 전달 프로토콜이다. 도메인 지식이 아닌 **Claude 세션 운영 가드**.

## 1. 근거

사용자 직접 피드백(2026-04-19):
- 1차 산출물은 마크다운으로 터미널에 표시되지만, 실무에서는 **PPT · Notion · PDF · 슬랙 · 이메일** 등 2차 산출물이 필요한 경우가 다수
- Claude가 "여기까지 결과입니다"로 마감하면 사용자가 매번 후속 변환을 재요청해야 함 → 단계 절약 필요
- 사용자가 매번 "어떤 포맷으로 뽑아달라"고 말하는 부담 제거가 목적

`.claude/rules/compounding.md` §1("2회+ 반복 승격")의 **예외**: 사용자 직접 지시는 1회 관찰로도 룰 승격 가능.

## 2. 핵심 원칙

### 2.1 1차 산출물은 마크다운 고정

- 플레이북 실행의 1차 결과는 **마크다운 파일** 필수 (`outputs/YYYY-MM-DD_{playbook}.md` 또는 `scenarios/{slug}/runs/YYYY-MM-DD_{playbook}/output.md`)
- 템플릿 구조·분량 가드·자가 체크리스트가 마크다운 기준으로 설계됨
- **1차를 건너뛰고 PPT·Notion으로 바로 가지 않는다** (검증 가능성 상실 + verifier 호출 불가)

### 2.2 마감 시점에 후속 포맷 선제 제안 (필수)

파일 저장 완료 → 파일 경로 안내 → **다음 형식으로 후속 포맷 옵션 제시**:

```
📄 산출물 저장: {경로}

후속 포맷이 필요하시면 어떤 형태로 변환해드릴까요?

1. **경영진 슬라이드** — PPT 초안 10-15장 (Exec Summary + 7원칙 + 우선순위 + 로드맵)
2. **Notion 문서** — toggle·체크박스·표 보존, DB 적재 고려
3. **PDF 인쇄용** — 요약 1p + 본문 5-10p + 부록 분리
4. **슬랙 공유** — mrkdwn 3~5단락, 핵심 발견·권고만
5. **이메일 보고서** — Exec Summary 본문 + 상세 첨부
6. **그대로 OK** — 추가 변환 없음

원하시는 번호를 말씀해주시거나, 다른 형태가 필요하면 알려주세요.
```

### 2.3 포맷별 변환 규약

| 포맷 | 변환 방식 | 보존 원칙 |
|------|-----------|-----------|
| PPT 슬라이드 | Exec Summary 1장, 7원칙 표 1장, 우선순위 매트릭스 1장, 로드맵 3장(Quick/구조/정교화), §6.5 정치 2-3장, 리스크·지표 각 1장 | 수치·출처 인용 유지, 시각화는 표 우선 |
| Notion 문서 | 섹션별 toggle, 표 원본, 체크박스 유지, 부록은 하위 페이지로 분리 | 회사 고유 정보 노출 여부 먼저 확인 |
| PDF | Pandoc 또는 mdBook 빌드 전제, 마크다운 구조 최대 보존 | 레이아웃 변동 최소화, 부록 페이지 분리 |
| 슬랙 mrkdwn | 핵심 발견 3개 + 권고 3단계 + 다음 액션만 | 600자 내외, @멘션 · 이모지 남용 금지 |
| 이메일 | Exec Summary 본문 + 상세 첨부 링크 | 수신자별 공개 범위 확인 후 발송 |

### 2.4 §6.5 정치 맥락 섹션 보호

`scope-discipline.md` §6.5(정치 맥락·공개 순서·리스크 로그)는 민감도가 높아 2차 포맷 변환 시 다음 규칙 적용:

| 포맷 | §6.5 처리 |
|------|----------|
| 내부 Notion / 내부 PDF | 원본 유지 가능 |
| PPT (경영진 대상) | **생략 옵션 질문 후 처리** — "이 섹션 포함할까요?" |
| 공개 PDF / 외부 공유 | **기본 제외**, 사용자 명시 요청 시에만 포함 |
| 슬랙 · 이메일 | **기본 제외**, 사용자 명시 요청 시에만 |

## 3. 금지 패턴

1. **무단 포맷 변환** — 사용자가 요청 안 했는데 PPT·Notion부터 만들기
2. **묻지 않고 끝내기** — 저장 후 "완료됐습니다" 한 줄로 마감 (옵션 제시 없음)
3. **옵션 나열 생략** — "어떻게 써드릴까요?"만 묻고 선택지 안 줌 → 선택 부담 사용자 전가
4. **§6.5 무시 변환** — 정치 맥락이 포함된 산출물을 민감도 안내 없이 공개 PPT로 변환
5. **경로 안내 누락** — 어느 파일에 저장됐는지 명시 안 함

## 4. 자가 체크 (산출물 마감 직전)

- [ ] 1차 마크다운 파일 저장 완료 (`outputs/` 또는 `scenarios/.../runs/`)
- [ ] 파일 경로를 사용자에게 명시적으로 안내
- [ ] 후속 포맷 6개 옵션 제시 (PPT / Notion / PDF / 슬랙 / 이메일 / 그대로)
- [ ] §6.5 정치 맥락 포함 시 민감도 안내 동반

4문항 모두 ✅ 후 대화 마감.

## 5. 롤백 조건

- **의무 피로**: 후속 포맷 옵션 제시가 모든 산출물에서 불필요하게 반복됨 (예: 사용자가 계속 "그대로 OK" 선택) → 누적 3회 이상 관찰 시 간소화 검토
- **오선택 증가**: 옵션이 6개라 오히려 선택 장애 유발 → 기본 3개 + "기타" 구조로 재설계

이 중 2건 이상 2회차 연속 관찰 시 본 규율 부분 개정.
