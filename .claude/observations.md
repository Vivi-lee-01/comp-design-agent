# Observations Log

본 파일은 **플레이북 실행 바깥**에서 발견된 메타 관찰을 기록한다. `scenarios/*/runs/*/evaluation.md`가 verifier 평가의 공식 입력이라면, 이 파일은 **사용자 대화·외부 레포 비교·임시 검토** 같은 경로에서 나오는 구조적 관찰의 입력구다.

## 기록 포맷

각 관찰은 다음 6항목으로 남긴다:

- **날짜**
- **출처**: 대화 / 외부 검토 / 기타
- **관찰**: 한 줄 요약
- **추정 해결 축**: 재료 / 설계 / 교정 / 루프 구조(메타)
- **재평가 조건**: 2회차 재발 시점 또는 특정 트리거 이벤트
- **상태**: 관찰 / 승격 검토 / 승격 / 기각

## 원칙

- **구조 결함**은 1회 관찰로도 보완 가능 — `.claude/rules/compounding.md` §1의 "2회+ 재발" 규율은 **산출물 품질 관찰용**이므로 루프 자체의 구조 누락에는 적용 기준이 다르다.
- **패턴·드리프트 관찰**은 §1에 따라 2회+ 재발 대기.
- 관찰 기록 후 재평가 조건이 오래 충족 안 되면 90일 기준 정리 대상으로 표기.
