# Playbook: Diagnose Current System

현 평가보상 제도를 진단하고 개선 우선순위를 도출하는 절차.

## 실행 조건

- `context/current-system.md` 가 충분히 채워져 있어야 함
- 가능하면 `context/compensation-data.md` 의 요약 통계 존재

## 산출물

- **Diagnosis Report** (`templates/diagnosis-report-template.md` 형식)
- 개선 우선순위 리스트

## 단계

### Step 1: 기준 로드

에이전트는 아래를 순서대로 읽는다:

1. `knowledge/principles.md` — 평가 기준점
2. `context/company-profile.md` — 조직 맥락
3. `context/current-system.md` — 진단 대상
4. `context/constraints.md` — 개선 여력

### Step 1-보강 (Scope 판정)

본 플레이북 진입 전 `.claude/rules/scope-discipline.md`의 §0 Scope 판정 5단계를 수행하라. 진단은 전체 제도를 스캔하지만 개별 이슈의 해결은 pay-band/leveling/review-cycle로 **이관**되므로, 4분류 중 "이관 인터페이스(→ 대상 task · 입/출력 계약)" 명시가 특히 중요한 플레이북이다.

### Step 2: 7원칙 점검

`principles.md`의 7가지 원칙을 기준으로 현 제도를 평가.

각 원칙별로:
- **현 상태** (한 문단)
- **점수** (1~5)
- **근거** (구체적 사례/증거)
- **개선 가능성** (High/Medium/Low)

| 원칙 | 점수 | 근거 | 개선 가능성 |
|------|------|------|-------------|
| 1. 명시적 철학 | | | |
| 2. 단순성 | | | |
| 3. 투명성 | | | |
| 4. 시장 연동 | | | |
| 5. 성과 연결 | | | |
| 6. 공정성 | | | |
| 7. 반복 가능성 | | | |

### Step 3: 안티패턴 스캔

`knowledge/principles.md` §안티패턴 **등재 전체(현 6개)**를 점검한다. 목록은 `principles.md` **최신 기준**이며, 아래는 현재 스냅샷 (확장 시 본 목록·`scenarios/rubric.md` 상단 의존성 박스 동기 갱신 — 절차: `.claude/rules/knowledge-sync.md`).

- [ ] Comp 먼저, Level 나중
- [ ] 협상 여지 과다 (동일 역할 내 30% 이상 격차)
- [ ] 평가 인플레이션 (모두 "우수")
- [ ] 원칙 없는 예외 다발
- [ ] 벤치마크 과신 또는 부재
- [ ] 분석만 있고 시정 없음 (analysis-without-action)

해당되는 안티패턴 있으면 **심각도 표기**.

### Step 4: 구조적 누락 확인

현 제도에서 **아예 없는** 핵심 요소:

- [ ] Pay Philosophy 명문화
- [ ] Pay Band
- [ ] Job Leveling
- [ ] Calibration 프로세스
- [ ] 정기 1on1 문화
- [ ] 시장 벤치마크 주기
- [ ] 페이 갭 분석
- [ ] 연간 리뷰 사이클

### Step 5: 데이터 기반 이슈 발굴

`compensation-data.md` 가 있다면:

- **밴드 이탈 비율**
- **동일 레벨 내 급여 편차 (최대-최소)**
- **페이 갭 존재 여부**
- **레벨 간 역전 현상**

### Step 6: 우선순위 도출

각 이슈에 대해:

**Impact × Urgency × Feasibility** 매트릭스로 우선순위.

| 이슈 | Impact | Urgency | Feasibility | 종합 |
|------|--------|---------|-------------|------|
| | H/M/L | H/M/L | H/M/L | |

**권장 우선순위 로직**:
1. **High Impact + High Urgency**: 즉시 착수
2. **High Impact + Low Urgency**: 분기 내 착수
3. **Low Impact + High Urgency**: 간단히 처리
4. **Low Impact + Low Urgency**: 백로그

매트릭스 아래에 **"선정 로직 주석"** 블록을 붙인다. 왜 이 순서로 묶였는지, 어떤 이슈가 다른 이슈의 선행조건인지, Feasibility가 낮은 이유(재원·정치·문화)를 2~5줄로 명시. 점수만으로 설명되지 않는 의사결정 맥락을 남겨야 이후 단계에서 누가 읽어도 재현·반박 가능하다.

### Step 6.5: 정치적 맥락·공개 순서 점검

우선순위 매트릭스만으로 로드맵을 짜면 제도가 정치적 현실에 충돌해 깨진다. 로드맵 착수 전에 다음을 점검한다:

**1) 저항 지도 (Resistance Map)**
- 각 권고별로 "누가 반대할 가능성이 큰가" 식별 (매니저 집단 / CEO 개인 / 특정 직군 등)
- `constraints.md`에서 "기존 시도·드롭 이력" 단서 우선 탐색 (예: "Calibration 2년간 2회 드롭 — 매니저 저항")
- 반대를 **피하지 않고 구조에 반영** — 예: 전면 도입 대신 파일럿, 강제 대신 가이드

**2) 공개 순서 (Disclosure Sequencing)**
- 투명성 관련 권고(밴드 공개, 분석 결과 공유 등)는 **선행조건이 해결된 후** 공개 원칙
- 전형적 잘못된 순서: 문제 있는 제도를 먼저 공개 → 신뢰 붕괴 → 개선 동력 상실
- 올바른 순서 예시: ① 정책/원칙 공식화 → ② 이상치 조정 or 사유 설명 → ③ 제도·데이터 공개

**3) 외부 강제력 활용**
- IPO·SOX·투자자 실사·공공 감사처럼 **조직 외부에서 오는 강제력**은 반대를 우회하는 레버. 권고에 엮을 수 있으면 엮는다.
- 강제력이 없는 조직에서는 "CEO 스폰서" 또는 "크리티컬 이슈 공론화(슬랙·이탈)"가 동등 기능.

**4) 리스크 로그**
- 각 권고에 대해 "이 권고가 정치적으로 깨지는 시나리오"를 1개씩 기술
- 리포트의 리스크 섹션(템플릿 §10)에 반영

이 Step은 **Hard 시나리오에서 점수 결정 요인**이다. 숫자 기반 권고만 쓴 리포트는 실행 단계에서 무력화된다.

**난이도별 분량 가이드**:
- **Easy 시나리오** (정치 변수 1~2개): 저항 지도 + 리스크 로그 2블록 축약. 공개 순서·외부 강제력은 해당 사항 있을 때만 1~2줄.
- **Medium** (정치 변수 3~4개): 4블록 모두 균형 전개.
- **Hard** (정치 변수 5+개, 문화 저항 누적): 4블록 모두 상세, 저항 지도에 stakeholder 3인 이상 개별 분석.

Easy에서 억지로 4블록을 "형식적 채우기"하지 말 것 — 시나리오 실제 정보에 맞물리지 않는 서술은 오히려 감점 요인.

### Step 7: 로드맵 제안

3~6개월 로드맵:

```
Month 1~2: [Quick win 중심]
Month 3~4: [구조 개선 중심]
Month 5~6: [정교화 중심]
```

각 단계별:
- 착수할 작업
- 필요 리소스
- 성공 지표

**재원 민감 권고 작성 시**: 밴드 재조정·인상 재원 배분처럼 재원 규모에 따라 실행 가능성이 달라지는 권고는 **단일 안이 아니라 3종 시나리오(재원 중립 / 재원 +N% / 재원 +2N%)로 제시**한다. 의사결정자(CFO·CEO)가 비교표를 근거로 선택할 수 있게 한다. 단일 안만 제시하면 "예산 없다"는 거부로 전체가 기각될 가능성이 커진다.

**재원 축 식별**: "재원"은 상황에 따라 복수 축을 가진다. 권고 작성 전 **지배적 재원 축**을 1차 식별하고, 해당 축 기준 3종 시나리오를 작성한다.
- **Cash**: 밴드 재조정·인상 폭 (대부분 기본값)
- **Equity**: Pre-IPO·초기 스타트업 Refresh Grant·옵션 풀
- **FX**: 글로벌 원격 조직 환율 버퍼
- **세후·EOR**: 다국적 직접 고용 vs EOR 비용 구조
- **단계 이행 경로**: 긴 로드맵(6~12개월)에서 중립→+N→+2N 전환 시점 명시

복수 축이 동시에 작동하는 컨텍스트(예: Pre-IPO = Cash + Equity, 글로벌 원격 = Cash + FX + 세후)는 **축별 3종 시나리오를 교차표로 제시**하거나 **지배적 축 1개 선택 + 보조축 민감도 주석**으로 처리한다. 단일 축만 제시하는 것이 부자연스러우면 독자는 다른 축의 제약을 읽어낼 수 없다.

### Step 8: 리포트 작성

`templates/diagnosis-report-template.md` 형식으로 산출물 작성.

**포함 요소**:
- Executive Summary (1페이지)
- 7원칙 점검 결과
- 안티패턴/누락 분석
- 데이터 기반 이슈
- 우선순위 매트릭스
- 로드맵
- 부록: 상세 근거

## 체크포인트

리포트 제출 전 자가 점검:

- [ ] 모든 주장에 **근거** (문서/데이터/관찰) 있는가?
- [ ] 추측과 사실이 **명확히 구분**되는가?
- [ ] 제약 (`constraints.md`) 을 위반하는 권고가 없는가?
- [ ] 개선안은 **실행 가능한 수준**으로 구체적인가?
- [ ] 민감 정보가 **식별 가능한 형태**로 노출되지 않는가?
